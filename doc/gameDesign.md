# Don't Let Me Go 小型游戏策划案

## 1. 项目定位

《Don't Let Me Go》是一款结合心理叙事、荒野求生与现实成长循环的独立游戏。

本项目是作者的第一款游戏开发学习项目，主要目标是通过 Unreal Engine、C++、VS Code 和 GitHub，完整经历一次从想法、策划、原型、开发、测试到迭代的游戏制作流程。

游戏不追求一开始就做成大型开放世界，而是先完成一个小而完整的可玩版本，让玩家能够体验到“现实中学习，梦境中实践，失败后成长，再次挑战”的核心循环。

## 2. 核心概念

玩家扮演一个在学习和生活中逐渐沉沦的学生。主角一边渴望不平凡的人生，一边又被现实中的压力、迷茫和无力感困住。

某天，主角梦见自己在前往海外留学的途中遭遇意外，流落到一座荒岛。起初，主角崩溃、恐惧、无助，但很快意识到自己必须想办法活下去。

主角尝试运用现实中学过的知识解决饮食、饮水、避难和安全问题。然而，由于知识和技能有限，主角会不断遭遇挫折，甚至在梦中失败。梦醒后，主角庆幸这只是一个梦，但也开始主动补充相关知识。

第二天，主角再次做梦，又回到了同一座荒岛。之前未完成的事业仍在那里等待着他。

游戏在现实与荒岛之间循环：现实中的学习会影响梦境中的生存能力，梦境中的失败也会推动现实中的成长。

## 3. 游戏主题

### 3.1 表层主题

- 荒岛求生
- 资源收集
- 制作工具
- 维持生命状态
- 在恶劣环境中寻找生路

### 3.2 深层主题

- 在沉沦中重新找回行动力
- 通过失败理解知识的价值
- 把幻想中的“不平凡生活”转化为现实中的一点点改变
- 学习不是抽象的任务，而是在关键时刻能够保护自己的能力
- 成长不是一次觉醒，而是一次次摔碎后继续尝试

## 4. 标题含义

《Don't Let Me Go》可以有多层含义：

- 不要让我继续沉沦下去
- 不要让我放弃自己
- 不要让我从这个梦中逃走
- 不要让我被荒岛吞没
- 不要让我失去继续成长的可能

标题既可以指主角对自己的呼喊，也可以指梦境、现实、过去的自己或未来的自己对主角的拉扯。

## 5. 类型与参考方向

### 5.1 游戏类型

- 第三人称生存游戏
- 轻度模拟经营
- 心理叙事游戏
- 循环成长类游戏

### 5.2 参考方向

- 《大多数》：现实压力、生活状态、普通人的困境
- 荒野求生题材：资源管理、饥饿、饮水、避难、工具制作
- Roguelite 式循环：失败后带着知识或经验再次尝试

本项目不会直接复刻任何参考游戏，而是吸收它们的结构特点，服务于“现实学习与梦境实践”的核心表达。

## 6. 玩家体验目标

玩家在游戏中应该感受到：

- 一开始的无助和混乱
- 资源不足时的紧张
- 失败时的挫败
- 学到新知识后的希望
- 下一次入梦时变强了一点点的成就感
- 现实和梦境互相影响的奇妙感

游戏的重点不是惩罚玩家，而是让玩家理解：失败不是终点，而是下一次准备的理由。

## 7. 核心玩法循环

游戏的核心循环如下：

1. 现实阶段：主角醒来，回顾上一次梦境中的失败或进展。
2. 学习阶段：主角获得新的知识、配方或生存理解。
3. 入梦阶段：主角再次来到荒岛。
4. 生存阶段：玩家探索、收集资源、制作工具、解决饥饿和口渴。
5. 挫折阶段：玩家可能因为知识不足、资源不足或决策失误而失败。
6. 梦醒阶段：梦境结束，部分经验转化为现实中的成长。
7. 下一轮循环：玩家带着新的知识再次进入荒岛，推进更远。

简化表达：

现实学习 -> 梦境实践 -> 遭遇失败 -> 梦醒反思 -> 获得成长 -> 再次入梦

## 8. 第一版可玩原型范围

第一版目标不是做完整游戏，而是做出一个能表达核心循环的小型原型。

### 8.1 第一版目标

完成一个“第一天荒岛梦境 + 梦醒 + 第二次入梦”的基础流程。

玩家能够：

- 控制角色在荒岛上移动
- 拾取基础资源
- 查看简单背包
- 制作一个基础生存物品
- 受到饥饿、口渴、体力、精神等状态影响
- 因失败或撑过一定时间而梦醒
- 在现实阶段获得一点新知识
- 再次进入同一座荒岛

### 8.2 第一版暂不制作

为了保证项目能推进，以下内容暂时不进入第一版：

- 大型开放世界地图
- 复杂建筑系统
- 复杂战斗系统
- 动物生态系统
- 天气系统
- 多结局剧情
- 大量 NPC
- 精致美术资产
- 完整存档系统

这些内容可以作为后续版本逐步加入。

## 9. 基础系统设计

### 9.1 角色控制系统

玩家可以控制主角在场景中移动、转向、跳跃和与物体互动。

第一版采用第三人称视角，方便观察角色状态和环境。

基础操作：

- 移动
- 跳跃
- 视角旋转
- 交互
- 打开背包
- 制作物品

### 9.2 资源系统

荒岛上存在基础资源，玩家靠近后可以拾取。

第一版资源：

- 树枝
- 石头
- 草
- 浆果
- 淡水

资源用途：

- 树枝：制作工具、火堆、庇护所
- 石头：制作工具、搭建火堆
- 草：制作绳子、简易铺垫
- 浆果：临时缓解饥饿，但可能存在风险
- 淡水：缓解口渴

### 9.3 背包系统

背包用于记录玩家持有的资源数量。

第一版只需要实现简单数量统计，不需要格子背包和重量系统。

示例：

- 树枝 x 3
- 石头 x 2
- 草 x 5

### 9.4 制作系统

玩家可以使用资源制作基础物品。

第一版制作配方：

- 简易工具：树枝 x 1 + 石头 x 1
- 火堆：树枝 x 3 + 石头 x 2
- 草绳：草 x 3

制作成功后，背包扣除资源并获得新物品。

### 9.5 生存状态系统

主角有基础生存数值，这些数值会随时间变化。

第一版状态：

- 饥饿
- 口渴
- 体力
- 精神

状态变化：

- 时间流逝会降低饥饿和口渴
- 跑步、采集和制作会消耗体力
- 夜晚、失败、受伤或长时间无进展会降低精神
- 进食、饮水、休息和成功制作物品会恢复部分状态

失败条件：

- 口渴降为 0
- 饥饿降为 0
- 精神降为 0

失败后不会真正死亡，而是梦醒。

### 9.6 梦境与现实切换系统

这是游戏最重要的特色系统。

梦境阶段：

- 玩家在荒岛上生存
- 可能失败，也可能完成阶段目标
- 梦境中的经历会形成学习线索

现实阶段：

- 玩家醒来
- 看到简短的反思文字
- 获得新的知识、配方或能力
- 可以再次入梦

第一版可以先用两个关卡或两个区域表现：

- Reality：现实房间
- IslandDream：荒岛梦境

### 9.7 知识成长系统

知识成长不是传统等级，而是“主角理解了什么”。

示例成长：

- 第一次梦中因缺水失败，醒来后获得“寻找淡水”的提示。
- 第二次入梦时，玩家更容易识别淡水资源。
- 第一次无法生火，醒来后获得“火堆配方”。
- 第二次入梦时，可以制作火堆。

第一版可以先实现最简单的知识解锁：

- 第一次入梦：只能收集资源，无法制作火堆
- 梦醒后：解锁火堆配方
- 第二次入梦：可以制作火堆

## 10. 第一版剧情流程

### 10.1 开始

主角处于现实房间中。房间略显杂乱，书本、电脑、行李和未完成的学习计划暗示主角正处在迷茫时期。

主角准备前往海外留学，但内心充满压力和逃避。

入睡后，梦境开始。

### 10.2 第一次梦境

主角醒在荒岛海滩。

周围只有海浪、树木、石头和散落的资源。

玩家尝试探索，学习如何移动、拾取资源和维持基础生存状态。

由于缺乏知识，主角很难真正改善处境。玩家可能因为饥饿、口渴或精神崩溃而梦醒。

### 10.3 第一次梦醒

主角从床上惊醒，意识到自己刚才只是做梦。

他感到庆幸，也感到后怕。

现实阶段出现一段反思：

“如果这不是梦，我可能真的活不过第一天。”

主角开始查找基础荒野求生知识。

解锁内容：

- 火堆配方
- 淡水识别提示

### 10.4 第二次梦境

主角再次回到同一座荒岛。

这一次，玩家可以利用上一次梦醒后学到的知识，尝试制作火堆，并撑过更长时间。

第二次梦境的目标：

- 收集足够资源
- 制作火堆
- 坚持到夜晚

完成后，第一版原型结束。

## 11. 美术与氛围方向

### 11.1 现实场景

现实场景应当压抑、安静、略显凌乱。

关键词：

- 狭小房间
- 昏暗灯光
- 未完成的计划
- 书本和电脑
- 行李箱
- 时间流逝感

### 11.2 梦境荒岛

荒岛场景不需要一开始很大，但需要有清晰的求生感。

关键词：

- 海滩
- 森林边缘
- 石头
- 枯枝
- 可收集资源
- 昼夜变化
- 远处未知区域

### 11.3 情绪变化

第一次梦境：

- 陌生
- 恐慌
- 混乱
- 无助

第二次梦境：

- 仍然紧张
- 但更有方向
- 玩家开始感觉“我可以试试”

## 12. 音效与音乐方向

第一版可以先使用临时音效。

需要的基础音效：

- 脚步声
- 海浪声
- 拾取资源声
- 制作成功声
- 状态危险提示声
- 梦醒音效

音乐方向：

- 现实中偏安静、压抑
- 梦境中偏环境音和低强度紧张感
- 梦醒时短暂突出心跳或呼吸感

## 13. 开发里程碑

### 13.1 阶段一：项目基础

目标：

- 确认 UE C++ 项目可以正常打开
- 确认 VS Code 可以编辑代码
- 确认 GitHub 记录正常
- 整理策划案和开发日志

完成标准：

- 项目能运行
- Git 能提交
- 文档结构清晰

### 13.2 阶段二：基础角色

目标：

- 创建或使用第三人称角色
- 实现移动、跳跃、视角控制
- 创建测试关卡

完成标准：

- 玩家可以在场景中自由移动

### 13.3 阶段三：交互与资源

目标：

- 创建可拾取资源
- 实现靠近交互
- 拾取后资源进入背包数据

完成标准：

- 玩家可以拾取树枝、石头、草

### 13.4 阶段四：背包与制作

目标：

- 实现简单背包
- 实现基础制作配方
- 制作火堆或简易工具

完成标准：

- 玩家能通过资源制作出至少一个物品

### 13.5 阶段五：生存状态

目标：

- 实现饥饿、口渴、体力、精神
- 状态随时间变化
- 状态归零触发梦醒

完成标准：

- 玩家需要管理基础生存状态

### 13.6 阶段六：现实与梦境循环

目标：

- 制作现实场景
- 制作荒岛场景
- 实现梦醒和再次入梦
- 实现知识解锁

完成标准：

- 玩家能完整体验一次失败、梦醒、学习、再次入梦的循环

## 14. 第一版验收标准

第一版原型完成时，应满足以下条件：

- 玩家可以从现实进入梦境荒岛
- 玩家可以在荒岛移动和探索
- 玩家可以拾取至少 3 种资源
- 玩家可以看到或感知自己的生存状态
- 玩家可以制作至少 1 个物品
- 玩家会因为失败或完成目标而梦醒
- 梦醒后会获得新的知识或配方
- 玩家可以再次进入荒岛，并使用新知识推进

只要以上内容成立，第一版就算成功。

## 15. 后续扩展方向

第一版完成后，可以逐步考虑：

- 更复杂的制作系统
- 简易庇护所搭建
- 雨水收集
- 生火失败概率
- 伤口、感染和疾病
- 植物识别
- 天气与昼夜
- 梦境记忆保留
- 现实学习安排
- 现实压力系统
- 更完整的剧情章节
- 多个梦境区域

## 16. 当前开发原则

为了保证项目能持续推进，当前阶段遵守以下原则：

- 先做能玩的，再做漂亮的
- 先做小循环，再做大世界
- 先做核心系统，再做复杂内容
- 每完成一个小功能就记录开发日志
- 每完成一个稳定节点就提交 Git
- 遇到不会的内容，优先拆小，而不是放弃

## 17. 一句话总结

《Don't Let Me Go》是一款关于“在梦境荒岛中学习生存，也在现实生活中学习活下去”的循环成长型荒野求生游戏。
